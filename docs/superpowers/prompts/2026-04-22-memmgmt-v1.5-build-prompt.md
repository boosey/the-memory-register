## 9. Known pitfalls (read once)

- **Playwright's `webServer` on Windows**: `pnpm build && pnpm start` in `webServer.command` may need a shell wrapper. If `webServer` startup fails with command-not-found, use `"cmd /c pnpm build && pnpm start"` on Windows.
- **Next.js `pnpm start` after standalone build**: the existing `postbuild` script copies standalone assets; do not break it. If `pnpm start` doesn't find the server, check `package.json` `"start"` script and confirm it points at the standalone output or a plain `next start`.
- **Monaco in E2E**: Monaco takes a moment to initialize. If a test asserts against Monaco content, `await page.locator(".monaco-editor").waitFor()` before interacting.
- **React Flow in E2E**: React Flow renders `canvas` elements for edges; DOM assertions against edges are flaky. Prefer `.react-flow__node` count assertions over `.react-flow__edge` for reliability.
- **Fixture backups**: the `03-edit-save-undo.spec.ts` test writes backups into the fixture's `the-memory-register-backups/` dir. The test's `finally` block cleans up. If a test aborts between write and cleanup, manually `rm -rf tests/fixtures/sample-claude-home/the-memory-register-backups` before retrying.
- **Parallel E2E workers**: the plan sets `workers: 1` intentionally because the edit-save-undo test mutates the fixture. Do NOT bump workers above 1 without isolating fixture writes.
